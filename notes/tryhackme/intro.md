# TryHackMe Introduction – Cybersecurity Roles

## Overview

Cybersecurity can broadly be divided into two main areas:

- Offensive Security (attacking systems to find weaknesses)
- Defensive Security (protecting systems from attacks)

---

## 🔴 Offensive Security

### Definition
Thinking like an attacker to identify vulnerabilities before real hackers exploit them.

### Key Focus
- Finding weaknesses in systems
- Exploiting vulnerabilities in a controlled and legal way

---

## 🔵 Defensive Security

### Definition
Protecting systems, networks, and data from cyber threats.

### Key Focus
- Detecting attacks
- Investigating suspicious activity
- Responding before damage occurs

---

# 🛡️ Security Analyst (Blue Team)

### Role
Security analysts act as the **digital defenders** of an organisation.

---

### Responsibilities
- Monitor systems and network activity
- Investigate alerts and suspicious behaviour
- Determine if an incident is a real threat
- Respond appropriately to security incidents

---

### Example Incident
- Employee logs in from London  
- Suddenly logs in from another country  
➡️ Requires investigation to confirm legitimacy

---

### Typical Day
- Monitor network and device activity
- Investigate unusual logins or behaviour
- Analyse events (what happened, when, how)
- Work with teams to improve security

---

### Career Path
- Entry-level role in defensive security
- Can specialise into:
  - Threat hunting
  - Incident response
  - Malware analysis

---

# 🏗️ Security Engineer

### Role
Designs and maintains systems that protect an organisation.

### Key Idea
Acts as the **architect of security systems**

---

### Example
- Maintaining an Intrusion Detection System (IDS)
  - Works like a “security camera” for networks
  - Detects suspicious or malicious activity

---

### Responsibilities
- Design and maintain security systems
- Stay updated on new attack techniques
- Assess risks and vulnerabilities
- Document procedures and security policies

---

### Career Path
- More specialised defensive role
- Can move into:
  - Cloud security
  - Application security
  - Offensive security

---

# 🧪 Penetration Tester (Ethical Hacker)

### Role
Tests how secure systems are by attempting to break into them legally.

---

### Key Idea
Simulates real attacks to find weaknesses before attackers do.

---

### Important Term
- **Engagement** → authorised agreement to test a system

---

### Responsibilities
- Test networks, systems, and applications
- Perform security assessments and audits
- Identify vulnerabilities
- Write reports with recommendations

---

### Typical Day
- Conduct controlled attacks
- Analyse vulnerabilities
- Report findings to organisations

---

### Career Path
- Entry into offensive security
- Can specialise into:
  - Web application testing
  - Network testing

---

## 🔴 Red Teaming (Advanced)

- Simulates full-scale, real-world attacks
- Long-term engagements
- May include physical access attempts

---

# 🧠 Key Takeaways

- Cybersecurity has two main sides:
  - Offensive (attack)
  - Defensive (protect)

- Security Analysts monitor and respond to threats  
- Security Engineers build protection systems  
- Penetration Testers simulate attacks  

- There are multiple career paths depending on interests:
  - Blue team (defensive)
  - Red team (offensive)
